The Basics

Web

  • A collection of HTML documents and other content (e.g. plain text, HTML, images, videos, music, etc.

HTML

  • Hypertext Markup Language
  • Text content - what you see
  • Markup - what it looks like
  • References to other documents (videos, etc.)
  • Links to other pages - connects multiple pages together

Major Pieces

  • The User Computer contains a program called a Browser that is used to interpret the HTML page and display it on the user's screen.
  • The Internet is the world's largest computer network.
  • HTTP protocol is the main protocol used on the web; this enables the user computer to send and receive HTTP Requests with a server.
  • Servers are computers that host the files that make up the Web. The server finds the appropriate HTML document and sends it back to the user's computer, where the web Browser interprets and dispays the page.
  • HTML Markup is made up of tags.
  • HTML ignores white space; you have to code line breaks, etc. to display the text the way you want it.
  • An Element refers to everything within a set of opening and closing tags.
  • An Attribute is a property of an HTML tag (e.g. HREF is an attribute of the anchor tag).
  • Tags can be inline or block; inline does not add any width or height; block tags surround the content in an invisible box, which adds height and width.
  • HTML document structure contains a heading (typically the title of the page); document type; an opening and closing "html" tag which surrounds the main body of the HTML content; and the body, which contains the main content of the page.

Common Tags

  • <b> - makes text bold (this should be last option to use for highlighting text
  • <strong> - used to define important text for screen readers (typically makes text bold);
  • <em> - makes text italic
  • <u> - makes text underlined
  • <a> - anchor - used for making links; contains attributes, e.g. HREF.
  • <img> - image - used to include images on the page; does not have a closing tag
  • <br> - inserts a line break; can have multiple; does not have a closing tag
  • <p> - paragraph - block tag; browsers automatically add some space (margin) before and after each element. The margins can be modified with CSS (with the margin properties).
  • <span> - used to group in-line elements in a document
  • <div> - block tag; defines a section in an HTML document and/or used to group block-elements to format them with CSS; very often used together with CSS, to layout a web page;
    • By default, browsers always place a line break before and after the element. However, this can be changed with CSS.
  • <h1-h6> - headers (largest to smallest font size)

Stages of Expertise

  • Ignorance - you don't know something exists; it takes about a minute to move from Ignornance to Awareness
  • Awareness - you now know something exists; it takes days to move from Awareness to Ability
  • Ability - you can do it, but takes some focus (not second nature); it could take years to move from Ability to Fluency
  • Fluency - you are a master/ expert at it (no thinking needed)

The Tools

  • Developer tools within the browser - e.g. Google Chrome allows you to view the HTML and CSS tagging
  • Multiple text editors exist to write your HTML:
    • Scratchpad - this is the most basic of editors; good for beginners but not as powerful, helps by adding closing tags, etc.; provides immediate feedback to the user so they can see their changes; unable to share your HTML (but you can write your HTML; and CSS (inside style elements, inside of Scratchpad - then copy paste into Codepen to share); only supports HTML and CSS languages
    • Codepen - ; provides feedback to the user in a few seconds so they can see their changes; can share your HTML but will require a Codepen account; supports HTML, CSS, and Javascript languages
    • Sublime - most advanced editor - ; requires a page refresh so the user so they can see their changes; able to share your HTML with other users (writes the code to your computer); you can drag the little icon at the top to a browser so you can see your HTML as a web page; supports all language; you can open an HTML file in the browser, and then make updates in Sublime, hit refresh in the browser to see the changes

The Languages and Structure

  • HTML - the language of the web; has syntaxes and rules; the browser turns HTML tags into elements that form a tree
  • DOM - document object model; tells the browser how to make the elements into the tree; the browser uses the DOM tree to display the web page
  • CSS - changes how the HTML elements look on a web page; CSS styles get assigned to classes as defined by the user; then these classes can be used as parameters in an HTML element to set the style
  • Javascript - used to code interactive items on a web page
  • Head tag - this is where you add the link to the CSS styling (using the "link" tag and specifying the name of the file that contains our CSS style information"

CSS Styling

  • The general rules to follow are
    • Get the structure of the page correct - look for natural boxes
    • Size and position the boxes on the page
    • Use Headings where appropriate. Look for repeated styles and semantic elements
    • Write the HTML
    • Apply styles from largest to smallest
    • Fix things - test in multiple browsers
  • Use Development tools to make and see changes to your code without having to change the CSS (initially)
  • It's critical to avoid repetition in coding for multiple reasons: code can become difficult to read; can lead to errors and/or inconsistencies in the code due to having to repeat the same code over and over again for elements which share the same styling characteristics; could make the code perform less efficiently for larger programs; and drive extra time and effort when changing and maintaining the code.
  • CSS styling enables us to avoid unnecessary repetition on the code
  • CSS rules cascade - the last or most specific rule takes precedence
  • CSS rules contain a Separator and a Declaration; you can use the same rule for multiple selectors by separating the selectors with commas
  • Declarations have two parts - property and value, and are separated by a semi-colon
  • CSS Selectors can be a Class name or a Tag name
  • CSS styling can be added in 3 different ways
    • Write the CSS in the <head> of the HTML using <style> syntax
    • Link the HTML to the separate CSS file; this is recommended and enables better organization for larger projects; the file is linked in the <head> of the HTML using < link rel and href>
    • Write the CSS inline with HTML - this is not recommended as it causes repetition of code
  • Browsers use default stylesheets to determine how to display HTML elements. Because rules differ sometimes, there are efforts to promote consistency in styles across browsers. One solution is using what is referred to as a CSS reset such as normalize.css
  • Box model - includes margin, border, padding, and content
    • Content - image or text appearing on the page (the nucleus)
    • Padding - clears an area around the content and the border of the box
    • Border - surrounds the box; is inherited from the color property
    • Margin - hidden space between the boxes (transparent)
  • Code should be structured and <div> is used to group/contain boxes. Use indentation in the code to help visually see the structure for easier reviewing
  • Boxes can have their size set by % or pixels; % allows the box to be resized based on the user screen
  • Postiioning of boxes can be done using the flex box layout within a division
